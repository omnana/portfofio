Technical Breakdown

Deep dive into the core architecture and visual engineering.

blur_on Dynamic Nebula Shader

The core challenge was rendering dense nebulae that could be flown through without significant performance drops. I implemented a custom ray-marching volume shader using HLSL that leverages local space signed distance fields (SDFs). 

// Fragment shader snippet for Nebula density
float4 frag(v2f i) : SV_Target {
    float3 rayPos = i.worldPos;
    float3 rayDir = normalize(rayPos - _WorldSpaceCameraPos);
    float density = 0;
    for(int j=0; j<MAX_STEPS; j++) {
        float d = map(rayPos);
        if(d < 0.01) {
            density += exp(-j * _StepSize);
        }
        rayPos += rayDir * _StepSize;
    }
    return float4(_NebulaColor.rgb, density);
}

Key Challenges & Retrospective

Developing Nebula Drift presented a unique hurdle: balancing visual fidelity with the extreme speeds required for the gameplay. Early builds suffered from motion sickness issues due to the way screen-space motion blur interacted with the procedural nebula volumes.

The solution involved decoupling the player's cockpit rendering from the external world blur using a dual-camera setup with specific layer masking. This kept the UI and ship interior crisp while allowing the outer space environments to streak dynamically.

Looking back, the project pushed my understanding of GPU optimization and custom pipeline development in Unity, ultimately resulting in a game that maintains a steady 144 FPS even in the densest asteroid fields.
